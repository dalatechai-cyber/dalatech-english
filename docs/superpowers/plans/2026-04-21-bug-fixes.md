# Bug Fixes — Streaming, Mobile Input, Facebook Share

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Fix three bugs: chunky AI text streaming, chat input hidden behind browser navigation on mobile, and broken Facebook certificate share.

**Architecture:** Streaming fix uses `flushSync` from `react-dom` to force per-chunk re-renders in React 18 (which otherwise batches async setState calls) plus anti-buffering server headers. Mobile fix replaces the broken `pb-3 pb-safe` CSS combo with a single `pb-input-area` utility that adds safe-area on top of base padding. Facebook fix replaces the broken `quote+u` URL with the supported `u`-only sharer URL, opening as a proper popup on desktop and direct navigation on mobile.

**Tech Stack:** Next.js 14 App Router, React 18, `react-dom/client` (flushSync), Tailwind CSS custom utilities

## Task 1: Fix streaming — anti-buffering headers + flushSync

**Files:**
- Modify: `src/app/api/chat/route.ts`
- Modify: `src/app/api/free-chat/route.ts`
- Modify: `src/components/ChatInterface.tsx`
- Modify: `src/components/FreeChatInterface.tsx`

**Root cause:** React 18 batches `setState` calls across `await` boundaries in async loops, so `setMessages` calls in the `while (true)` streaming loop may be coalesced — making text appear in chunks. The server also needs `Cache-Control: no-transform` to tell proxies not to buffer.

- [ ] **Step 1: Update response headers in `src/app/api/chat/route.ts`**

Find the `return new Response(readable, { headers: { ... } })` block and replace it:

```typescript
  return new Response(readable, {
    headers: {
      'Content-Type': 'text/plain; charset=utf-8',
      'Cache-Control': 'no-cache, no-store, no-transform',
      'X-Accel-Buffering': 'no',
      'X-Content-Type-Options': 'nosniff',
    },
  })
```

- [ ] **Step 2: Apply same header change to `src/app/api/free-chat/route.ts`**

Read the file, find its `return new Response(readable, { headers: { ... } })` and apply the same replacement as Step 1.

- [ ] **Step 3: Add `flushSync` to `src/components/ChatInterface.tsx`**

Add import at the top of the file:
```typescript
import { flushSync } from 'react-dom'
```

Then in the streaming `while (true)` loop, find:
```typescript
        setMessages(prev =>
          prev.map(m => (m.id === aiMsgId ? { ...m, content: fullContent } : m))
        )
```

Replace with:
```typescript
        flushSync(() => {
          setMessages(prev =>
            prev.map(m => (m.id === aiMsgId ? { ...m, content: fullContent } : m))
          )
        })
```

- [ ] **Step 4: Apply same `flushSync` fix to `src/components/FreeChatInterface.tsx`**

Add `import { flushSync } from 'react-dom'` at the top, then find and replace the `setMessages` call inside the `while (true)` loop with the `flushSync`-wrapped version (identical to Step 3).

```bash
git add src/app/api/chat/route.ts src/app/api/free-chat/route.ts src/components/ChatInterface.tsx src/components/FreeChatInterface.tsx
git commit -m "fix: smooth streaming — flushSync per-chunk + anti-buffering headers"
```

## Task 2: Fix mobile chat input hidden behind browser navigation bar

**Files:**

- Modify: `src/components/ChatInterface.tsx`
- Modify: `src/components/FreeChatInterface.tsx`

**Root cause:** The input section uses `pb-3 pb-safe` where `.pb-safe` **overrides** Tailwind's `pb-3` (last class wins), so on devices where `env(safe-area-inset-bottom)` is `0px`, padding drops to zero. The fix is a single utility that **adds** safe-area on top of the base 12px.

- [ ] **Step 1: Add `.pb-input-area` utility to `src/app/globals.css`**

Append after the existing `.pb-safe` rule:
```css
/* Input area: base padding + iOS/Android safe area inset */
.pb-input-area {
  padding-bottom: calc(0.75rem + env(safe-area-inset-bottom, 0px));
}
```

- [ ] **Step 2: Update `src/components/ChatInterface.tsx`**

```typescript

```


## Task 3: Fix Facebook certificate share

**Files:**
- Modify: `src/components/CertificateModal.tsx`

**Root cause:** The current URL uses `?quote=TEXT&u=URL` — the `quote` parameter is not reliably supported by the Facebook sharer. On mobile, `window.open(..., '_blank')` gets blocked by the browser. The fix uses only `?u=URL` (the reliable parameter) and opens as a named popup on desktop, direct navigation on mobile.

- [ ] **Step 1: Replace `handleShare` in `src/components/CertificateModal.tsx`**

Read the file first. Find the current `handleShare` function and replace it entirely with:

```typescript
  const handleShare = () => {
    const appUrl = 'https://core-english.vercel.app'
    const shareUrl = `https://www.facebook.com/sharer/sharer.php?u=${encodeURIComponent(appUrl)}`
    const isMobile = /iPhone|iPad|iPod|Android/i.test(navigator.userAgent)
    if (isMobile) {
      window.location.href = shareUrl
    } else {
      window.open(shareUrl, 'fb-share-dialog', 'width=626,height=436,toolbar=0,status=0,menubar=0,scrollbars=yes,resizable=yes')
    }
  }
```
